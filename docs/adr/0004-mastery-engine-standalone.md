# ADR-0004: 掌握度引擎独立部署

**状态**：已采纳 · **日期**：2025-04-20 · **提出者**：Architect

## 背景

掌握度建模（Bayesian Knowledge Tracking、遗忘曲线、来源权重）是一套独立的时序分析逻辑。它不依赖主应用的试卷分析管线，且可能有独立的性能需求和部署节奏。

## 决策

将掌握度引擎作为独立组件，通过磁盘 SQLite 文件与主应用交互：

```
主应用（Flask）       掌握度引擎（独立）
    │                      │
    │  ── 写入事件 ──→     │
    │  ←── 查询结果 ──     │
    │                      │
    mastery.db (SQLite)
```

架构拆分：

- `mastery_engine.py` — 核心引擎（BKT 更新、遗忘曲线、来源权重聚合），无 Web 依赖
- `mastery_api.py` — Flask API 包装层，提供 REST 查询接口
- `backend/application/mastery_service.py` — 主应用中的封裝，通过事件记录和查询调用

## 后果

正面：
- 引擎可独立测试和部署
- BKT 计算不影响主应用性能
- 数据通过 SQLite 文件传递，无网络 RPC

负面：
- SQLite 不能承载高并发写
- 共享文件路径需要部署时对齐
- 缺少事件溯源机制，掌握度变更记录在主应用和引擎两端分散存储
