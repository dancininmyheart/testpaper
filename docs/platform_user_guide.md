# PaperFlow 平台使用文档

本文档面向教师和平台试用人员，说明如何在当前平台中完成“上传试卷 -> 提取试题 -> 准备参考答案 -> 批改学生答题卡 -> 查看报告 -> 生成相似试卷”的完整流程。

## 1. 平台定位

PaperFlow 是一个大模型试卷智能分析与评分平台，当前核心能力包括：

- 创建试卷分析项目，管理学科、年级和项目状态。
- 上传试卷图片，使用 MinerU 与大模型提取结构化试题。
- 审查并确认试题内容、题号、题型、分值和知识点。
- 在“准备参考答案”阶段上传标准答案图片，或由 AI 基于试卷生成参考答案。
- 上传学生答题卡图片，识别作答并生成评分结果。
- 审查识别和评分结果，发布项目报告。
- 基于已分析试卷生成相似试卷，并下载 PDF 或 Markdown。

## 2. 启动与登录

### 2.1 启动平台

在项目根目录运行：

```bash
python run_fullstack_dev.py
```

默认访问地址：

```text
http://127.0.0.1:8020
```

也可以直接启动平台服务：

```bash
python run_platform_server.py --host 0.0.0.0 --port 8020
```

健康检查地址：

```text
http://127.0.0.1:8020/health
```

### 2.2 默认账号

首次启动时，系统会自动确保默认账号存在：

| 角色 | 用户名 | 默认密码 |
| --- | --- | --- |
| 管理员 | `admin` | `admin123` |
| 教师 | `teacher` | `teacher123` |

本地试用可以直接使用默认账号。正式环境应通过环境变量修改默认账号和密码。

## 3. 页面导航

登录后左侧导航包含以下入口：

| 页面 | 用途 |
| --- | --- |
| 工作台 | 查看项目总数、进行中项目、已完成项目和最近项目 |
| 项目 | 创建、打开、删除试卷分析项目 |
| 学生管理 | 新增和维护学生档案，供答题卡匹配使用 |

| 报告中心 | 查看已完成项目的报告 |

## 4. 核心概念

### 4.1 项目

项目是一套试卷从提取、参考答案、学生作答识别到报告生成的工作空间。每个项目包含：

- 项目名称、学科、年级。
- 上传的试卷图片。
- 提取后的试题列表。
- 参考答案。
- 学生答题卡分析记录。
- 最终报告和相似试卷。

### 4.2 试卷图片

试卷图片用于提取题目。当前前端上传控件接受图片文件，建议使用 JPG、PNG 或浏览器可识别的图片格式。

### 4.3 参考答案

参考答案用于后续批改学生答题卡。当前平台有两种准备方式：

- 上传标准答案图片或扫描件，由系统提取参考答案。
- 不上传答案，直接由 AI 基于已确认试题生成参考答案。

注意：标准答案图片不在“上传试卷图片”阶段上传，而是在项目进入“准备参考答案”后上传。

### 4.4 学生与答题卡

学生答题卡必须匹配一个已录入的学生。上传答题卡前，应先在“学生管理”中新增学生档案。

## 5. 完整使用流程

### 5.1 创建项目

1. 进入“项目”页面。
2. 点击“新建项目”。
3. 填写项目名称，例如“初三数学期中试卷”。
4. 选择年级，确认提取方式。
5. 点击“创建项目”，系统会进入该项目工作台。

当前提取方式以 MinerU 智能提取为主，适合复杂排版、公式和含图表的试卷。

### 5.2 上传试卷图片

进入项目后，在“阶段一 · 准备”中：

1. 点击或拖入试卷图片。
2. 可一次选择多张图片。
3. 确认文件数量后，点击“上传文件并保存”。

建议：

- 按试卷页码顺序上传。
- 图片尽量清晰、完整、无明显遮挡。
- 避免大角度倾斜、强反光、阴影覆盖公式或图形。
- 不要在此阶段上传标准答案；标准答案在后面的“准备参考答案”阶段处理。

### 5.3 提取试题

上传试卷后，在“阶段二 · 提取与审查”中点击“MinerU 深度视觉分析”。

系统会按以下步骤运行：

1. MinerU 解析试卷图片，识别文字和图片区域。
2. LLM 整理 OCR 结果，形成结构化题目列表。
3. AI 分析知识点。
4. VLM 匹配题目配图。
5. 保存提取结果。

处理期间页面会显示进度。复杂试卷可能需要较长时间，尤其是页数较多、公式较多或图表较多时。

### 5.4 审查并确认试题

试题提取完成后，项目进入“待审查试题”状态。

点击“审查并核对试题”，在试题审查页完成以下检查：

- 左侧查看题目列表和题号。
- 中间查看原始试卷图片。
- 右侧查看题干、题型、分值、知识点和匹配图片。
- 如项目处于待审查状态，可编辑题干内容。

确认无误后，点击确认按钮提交。提交后，项目会进入“准备参考答案”相关状态。

### 5.5 准备参考答案

试题确认后，如果项目还没有参考答案，工作台会显示“准备参考答案”区域。

有两种方式：

| 方式 | 适用场景 | 操作 |
| --- | --- | --- |
| 上传标准答案图片 | 已有教师答案、标准答案页或答案扫描件 | 选择“参考答案图片 / 扫描件”，点击“上传并开始解析参考答案” |
| AI 生成参考答案 | 没有标准答案，或希望快速生成初稿 | 点击“由 AI 生成参考答案” |

上传标准答案后，系统会尝试从答案图片中提取每道题的答案。如果有部分题目未提取到，系统会尝试用 AI 补齐，并在审查页显示来源标签。

### 5.6 审查参考答案

参考答案处理完成后，项目进入“审查参考答案”状态。

点击“校验并审核参考答案”，逐题检查：

- 题干内容。
- 解题分析。
- 解题步骤。
- 最终答案。
- 答案来源标签。

答案来源标签含义：

| 标签 | 含义 |
| --- | --- |
| 上传答案提取 | 来自上传的标准答案图片 |
| AI 补齐 | 上传答案未覆盖该题，系统用 AI 补齐 |
| AI 生成 | 未上传标准答案，由 AI 直接生成 |
| 未提取 | 当前题没有可用参考答案，需要人工重点检查 |

确认无误后，点击“全部确认无误”。项目会回到“就绪”状态，之后可以上传学生答题卡。

### 5.7 管理学生

在上传答题卡前，进入“学生管理”页面新增学生：

1. 输入学号。
2. 输入姓名。
3. 输入年级或班级。
4. 点击“录入学生档案”。

已创建的学生会出现在项目工作台的学生下拉框中。

### 5.8 上传学生答题卡并识别

当项目状态为“就绪”且已有参考答案时，在“阶段三 · 批改”中：

1. 选择对应学生。
2. 上传该学生的答题卡图片。
3. 点击“上传并开启答题卡智能识别”。

系统会识别学生作答、对照参考答案评分，并生成该学生的作答明细。运行记录会显示在“学生批改运行记录”中。

### 5.9 审查识别与评分结果

答题卡识别完成后，根据项目状态进入审查：

- “待审查识别结果”：点击“核对答题卡识别结果”。
- “待审查作答”：点击“审查对错判定”。
- 单个学生运行记录有结果时，可点击“查看作答明细”。

在作答审查页中检查识别内容、判分和报告数据。确认无误后点击：

- “确认识别结果并发布”，或
- “确认评分并发布”。

确认后，学生报告和项目数据会被更新。

### 5.10 查看报告

项目完成后，可以从两个入口查看报告：

- 在项目工作台点击“查看学情统计报告”。
- 在“报告中心”打开已完成项目。

报告会展示项目维度的分析结果。学生维度报告可以从学生详情或对应学生项目记录进入。

### 5.11 生成相似试卷

当项目已经完成提取与参考答案准备后，工作台会出现“生成相似试卷”按钮。

点击后，系统会基于原试卷的题型、知识点和结构生成新试卷。生成完成后：

- 可以在页面中预览试卷内容。
- 可以下载 PDF。
- 可以下载 Markdown。
- 可以点击“返回批改与分析”回到项目主流程。

## 6. 模型模式说明

项目工作台和参考答案审查页都提供模型模式选择。

| 模式 | 当前配置 | 特点 |
| --- | --- | --- |
| 快速模式 | `openai_vision_gemini_3_flash` + `openai_text_gemini_3_flash` | 更快，适合快速预览或简单试卷 |
| 标准模式 | `openai_vision_gemini_3_1_pro` + `openai_text_gemini_3_1_pro` | 更稳，适合正式分析、复杂公式和多图试卷 |

建议：

- 初次跑通流程可以使用快速模式。
- 正式批改、复杂数学试卷、参考答案质量要求较高时使用标准模式。
- 如果快速模式出现漏题、答案不完整或识别不稳定，可切换标准模式重新运行对应阶段。

## 7. 项目状态说明

| 状态 | 含义 | 用户动作 |
| --- | --- | --- |
| `draft` | 草稿，项目刚创建 | 上传试卷图片 |
| `extracting` | 正在提取试题 | 等待完成，必要时查看后端日志 |
| `review_questions` | 待审查试题 | 进入试题审查页确认 |
| `generating_answers` | 正在准备参考答案 | 等待上传答案提取或 AI 生成完成 |
| `review_answers` | 待审查参考答案 | 进入参考答案审查页确认 |
| `ready` | 项目就绪 | 上传学生答题卡，或生成相似试卷 |
| `recognizing` | 正在识别学生答题卡 | 等待识别和评分完成 |
| `review_recognition` | 待审查识别结果 | 核对答题卡识别结果 |
| `review_scores` | 待审查评分结果 | 审查并确认对错判定 |
| `completed` | 已完成 | 查看报告 |
| `generating_paper` | 正在生成相似试卷 | 等待生成完成 |
| `paper_ready` | 相似试卷已生成 | 预览、下载或返回批改流程 |
| `error` | 阶段运行失败 | 查看错误信息，修正后重试 |

## 8. 文件上传建议

### 8.1 试卷图片

- 每页尽量完整，不裁掉题号、图形、选项或页眉页脚中的关键信息。
- 多页试卷按页码顺序上传。
- 数学公式、几何图形、表格题尽量使用高清图片。
- 避免把答案页和试卷页混在一起上传到“试卷图片”区域。

### 8.2 标准答案图片

- 在“准备参考答案”阶段上传。
- 图片中最好保留题号，便于系统与试题列表对齐。
- 如果答案只写最终结果，没有过程，系统可能只能提取最终答案。
- 如果一张图包含多道题答案，建议题号清晰、排版分隔明显。

### 8.3 学生答题卡图片

- 先选择学生，再上传该学生对应的答题卡。
- 多页答题卡按作答顺序上传。
- 避免多个学生的答题卡混在同一次上传中。
- 字迹、批注、涂改过多时，应重点审查识别结果。

## 9. 常见问题

### 9.1 上传标准答案后为什么看不到选择文件？

当前产品流程中，标准答案不在第一阶段上传。第一阶段只上传试卷图片。标准答案应在试题确认后，在“准备参考答案”区域选择并上传。

### 9.2 上传标准答案后为什么一直没有结果？

可能原因：

- 项目正处于 `generating_answers`，后台仍在处理。
- 标准答案图片页数多、内容复杂，模型处理时间较长。
- 后端服务、模型服务或 MinerU 服务异常。
- 当前页面状态未刷新。

处理建议：

1. 等待一段时间后刷新项目页。
2. 查看项目状态和错误提示。
3. 检查后端日志是否有模型超时、解析失败或网络错误。
4. 若刚修改过代码，请重启 `python run_fullstack_dev.py` 后再试。

### 9.3 部分题目的参考答案为空怎么办？

常见原因：

- 标准答案图片里没有对应题号。
- 答案图片不清晰，题号或答案区域无法识别。
- 试题提取出的题号与答案图片中的题号不一致。
- 标准答案只覆盖部分题目。
- 复杂公式、图形题或步骤题导致提取失败。

处理建议：

- 在参考答案审查页查看来源标签。
- 对“未提取”的题目人工重点核对。
- 如果显示“AI 补齐”，说明上传答案未覆盖该题，系统已尝试生成补充答案。
- 必要时切换标准模式重新上传或重新生成参考答案。

### 9.4 为什么不能上传学生答题卡？

需要同时满足：

- 项目已进入 `ready`。
- 参考答案已确认。
- 学生管理中已有学生。
- 工作台中已选择学生。
- 已选择答题卡图片。

### 9.5 项目卡在某个状态怎么办？

先判断卡住的状态：

- `extracting`：试题提取阶段可能仍在运行，重点检查 MinerU 和模型日志。
- `generating_answers`：参考答案阶段可能仍在运行，重点检查标准答案图片和模型日志。
- `recognizing`：答题卡识别和评分阶段可能仍在运行，重点检查学生答题卡图片和后台任务日志。
- `generating_paper`：相似试卷生成按题处理，题量越多越慢。

如果状态长时间不变，并且后台没有继续输出，应检查后端是否仍在运行，再根据错误信息重试对应阶段。

## 10. 数据与产物位置

默认数据位置：

| 内容 | 默认路径 |
| --- | --- |
| 平台 SQLite 数据库 | `outputs/platform/app.db` |
| 上传文件与平台存储 | `outputs/platform/storage` |
| 模型配置 | `llm_config.json` |
| 关键词/知识点配置 | `key_word.json` |
| 学情数据库 | `mastery.db` |

不要随意删除 `outputs/platform` 下的数据，否则项目、上传文件、报告或运行记录可能丢失。

## 11. 推荐操作顺序

正式使用时建议按以下顺序执行：

1. 启动平台并登录。
2. 在“学生管理”中录入学生。
3. 创建项目。
4. 上传试卷图片。
5. 运行 MinerU 深度视觉分析。
6. 审查并确认试题。
7. 上传标准答案图片，或由 AI 生成参考答案。
8. 审查并确认参考答案。
9. 选择学生并上传答题卡。
10. 审查识别和评分结果。
11. 查看报告。
12. 如需要，生成相似试卷并下载。

## 12. 使用注意事项

- 每个阶段完成后都建议进入审查页确认，避免错误传递到后续批改。
- 参考答案是评分的核心依据，正式批改前必须重点核对。
- 快速模式适合试跑，标准模式适合正式分析。
- 标准答案上传与 AI 生成都可能存在不确定性，最终结果以教师审查确认为准。
- 对于数学、物理等公式密集试卷，应优先保证图片清晰度。
- 如果项目用于正式归档，保留原始图片、审查记录和最终报告。
